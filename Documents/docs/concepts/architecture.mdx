# Architecture

LangChainは、複数のパッケージで構成されるフレームワークです。

## langchain-core

このパッケージは、さまざまなコンポーネントの基本的な抽象化と、それらを組み合わせる方法を提供します。  
チャットモデル、ベクトルストア、ツールなどの主要なコンポーネントのインターフェースがここで定義されています。  
ここにはサードパーティの統合は含まれていません。依存関係は非常に軽量です。

## langchain

メインの`langchain`パッケージには、アプリケーションの認知アーキテクチャを構成するチェーンと検索戦略が含まれています。  
これらはサードパーティの統合ではありません。  
このパッケージ内のすべてのチェーン、エージェント、検索戦略は特定の統合に特化したものではなく、すべての統合で汎用的に使用できます。

## Integration packages

人気のある統合には、それぞれ独自のパッケージがあります（例：`langchain-openai`、`langchain-anthropic`など）。  
これにより、適切にバージョン管理され、適切な軽量化が可能になります。

詳細については、以下を参照してください：

* [integrations packages](/docs/integrations/providers/)
* 各統合パッケージに関する詳細情報が記載された [API Reference](https://python.langchain.com/api_reference/)

## langchain-community

このパッケージには、LangChainコミュニティによって維持されているサードパーティの統合が含まれています。  
主要な統合パッケージは分離されています（上記を参照）。  
このパッケージには、さまざまなコンポーネント（チャットモデル、ベクトルストア、ツールなど）の統合が含まれています。  
パッケージをできるだけ軽量に保つために、このパッケージのすべての依存関係はオプションです。

## langgraph

`langgraph`は、`langchain`を拡張したもので、グラフ内のエッジとノードとしてステップをモデリングすることで、LLMを使用した堅牢でステートフルなマルチアクターアプリケーションを構築することを目的としています。

LangGraphは、一般的なタイプのエージェントを作成するための高レベルのインターフェースと、カスタムフローを構成するための低レベルAPIを提供します。

:::info[Further reading]

* LangGraphの概要は [こちら](https://langchain-ai.github.io/langgraph/concepts/high_level/#core-principles)をご覧ください。
* LangGraph Academyコースは[こちら](https://academy.langchain.com/courses/intro-to-langgraph)をご覧ください。 

:::

## langserve

LangChainのチェーンをREST APIとしてデプロイするためのパッケージです。  
プロダクション対応のAPIを簡単に立ち上げることができます。

:::important

LangServeは、シンプルなRunnableをデプロイし、langchain-core内の既知のプリミティブと連携するために設計されています。

LangGraphのデプロイメントオプションが必要な場合は、LangGraphアプリケーションのデプロイに適したLangGraph Cloud（ベータ版）を検討するべきです。

:::

詳細については、[LangServe documentation](/docs/langserve)をご覧ください。 

## LangSmith

LLMアプリケーションをデバッグ、テスト、評価、監視するための開発者向けプラットフォームです。

詳細については、[LangSmith documentation](https://docs.smith.langchain.com)をご覧ください。
