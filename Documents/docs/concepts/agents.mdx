# エージェント

単独では、言語モデルは行動を実行することはできません。ただテキストを出力するだけです。  
エージェントとは、高度なタスクを受け取り、LLMを推論エンジンとして使用して、どの行動を取るかを決定し、その行動を実行するシステムです。

[LangGraph](/Documents/docs/concepts/architecture#langgraph) は、LangChainを拡張したもので、高度に制御可能でカスタマイズ可能なエージェントを作成することを目的としています。エージェントを構築する際には、LangGraphの使用をお勧めします。

詳細については、以下のリソースをご覧ください：

* LangGraphの[一般的なエージェントアーキテクチャ](https://langchain-ai.github.io/langgraph/concepts/agentic_concepts/)に関するドキュメント 
* [LangGraphのプリビルトエージェント](https://langchain-ai.github.io/langgraph/reference/prebuilt/#langgraph.prebuilt.chat_agent_executor.create_react_agent)

## レガシーエージェントの概念: AgentExecutor

LangChainは以前、エージェントのランタイムとして`AgentExecutor`を導入しました。  
これは優れた出発点となりましたが、より高度でカスタマイズされたエージェントに対応する際の限界が明らかになりました。  
そのため、現在では`AgentExecutor`を段階的に廃止し、LangGraph内のより柔軟なソリューションに移行しています。

### AgentExecutorからLangGraphへの移行

現在`AgentExecutor`を使用している場合でも、心配はいりません！以下のリソースをご用意しています：

1. `AgentExecutor`を引き続き使用する必要がある場合、[AgentExecutorの使用方法](/Documents/docs/how_to/agent_executor)に関する包括的なガイドを提供しています。

2. しかし、より柔軟で制御可能なLangGraphへの移行を強くお勧めします。この移行を容易にするため、`AgentExecutor`からLangGraphにスムーズに移行するための詳細な[移行ガイド](/Documents/docs/how_to/migrate_agent)を作成しました。
